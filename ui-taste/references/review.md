# review — 自评、评审与收工判定

两种分永不平均：**机械分**（`scripts/ui_taste check`，确定性，可挂 CI）与**判断分**（本文，人/模型裁量）。平均会掩盖哪部分是可复现的。

## 第 0 步：先看见再评（强制）

能用浏览器/截图工具就截三视口 + 暗色模式：桌面 1280×800、平板 768×1024、手机 375×812。一张截图值一千 token——大量问题（文字溢出、对比度、层级失衡）只在渲染后可见。**没有任何视觉工具就在结论里明说"未见渲染效果，以下仅基于代码"**，不许装作看过。

## 六镜头（按序过，产物是 findings 不是改写）

1. **anti-slop**：对照 anti-slop.md 分级清单 + 机械计数（眉标数 ≤ceil(节数/3)、zigzag ≤2、布局家族不重复、hero 文字元素 ≤4、CTA 意图不重复）。
2. **手艺**：craft.md 抽查——对比度、焦点、alt、表单、长内容、dvh。
3. **层级**：眯眼测试（模糊 8px）视线是否先落在唯一一处；相邻层级 ≥1.5×；≤4 个层级；≥3 档表面海拔。
4. **清晰**：每个元素回答"用户此刻要什么"；文案逐句重读（语法/指代/AI 腔）。
5. **签名与立意**：本页的唯一记忆点在哪？没有=平庸；两个=互相抵消。再做概念两测：
   盖住所有文字和 logo 还认得出主题吗？换个客户/产品名还成立吗（成立=模板）？
   **概念不及格时执行分不能救总评**——再精致也拉不回一个没有 idea 的设计。
6. **差距**：距离该品类最好的产品还差什么（具体指名，如"对比 Linear 的设置页，这里…"）。

## 启发式打分（要给分数时用）

Nielsen 十项各打 1-5：1=阻断任务 / 2=严重摩擦 / 3=能用但困惑 / 4=可用差润色 / 5=业界最佳。
六条设计定律各判 PASS/FAIL：Fitts（触达 ≥44px 移动 / ≥24px 桌面）· Hick（导航每层 ≤7；无未过滤的 20+ 选项 select）· Doherty（感知响应 <400ms、骨架 200ms 内出现）· Cleveland-McGill（时间序列用线/面积不用竖条与饼）· Miller（表单 ≤5 节、向导 ≤5 步）· Tesler（复杂度被默认值吸收而不是暴露成 8 个开关）。

```
UsabilityScore = clamp( round((mean(nielsen)-1)/4*100) - 5×FAIL数, 0, 100 )
A≥90 B≥80 C≥70 D≥60 F<60
```

每条 finding 打**一个**影响标签，按序排列：`blocks-conversion > adds-friction > reduces-trust > minor-polish`。
反馈层级过滤：Value（做的事对不对）不及格时，不要在报告里塞 Delight 级建议。

## 完工十关（收工前全过）

| 关 | 判据 |
| --- | --- |
| 1 层级 | 首屏视线唯一落点；相邻层级 ≥1.5× |
| 2 字系 | 视口内 ≤3 字重；无一次性字号 |
| 3 表面 | ≥3 档海拔（亮度差 ≥2% 浅色 / ≥4% 暗色） |
| 4 间距 | 组内<组间<节间；无随手 magic number |
| 5 图标 | 一族、一个 strokeWidth、与字重匹配 |
| 6 状态 | 八格按场景覆盖（components.md） |
| 7 动效 | 预算内、退场 75%、reduced-motion 处理 |
| 8 微文案 | 无占位文案、按钮动词具体、错误给出路 |
| 9 像素诚实 | 无文字溢出、无假内容冒充真内容、双模式都看过 |
| 10 数据格式 | tabular-nums、`—` 占缺、Intl 格式化 |

## 收工判定

```
BLOCKED   = 存在任何 Critical（一眼 AI / 阻断任务 / 无障碍硬伤）
NOT READY = 无 Critical 但有 Major
READY     = 零 Critical 零 Major（Minor 需明示接受）
```

- **零 finding = 评审失效信号**，不是通过：换镜头重跑一遍，第二遍仍为零才可作为通过证据。
- 评审输出两用：给人看用 `file:line - 一句话` 简洁式（可点击、按文件分组、干净文件写 `✓ pass`）；改造对照用 `| Before | After | Why |` 三列表。
- 最后一问（评审版换牌测试）：**"要把它从'AI 生成'提到'有人设计过'，最值得改的三处是什么？"**——答案就是下一轮工单。
