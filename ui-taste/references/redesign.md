# redesign — 改造存量页面

误判模式是烂改版的头号来源。先分类，只在真歧义时问一次："保留现有品牌演进，还是视觉推倒重来？"

- **保留式**：品牌不变求现代化。审计先行，提取既有 token，渐进演化。
- **推翻式**：内容与信息架构保留，视觉当绿地做（走 direction.md）。
- **绿地**：品牌本身在换。整套按新建流程走。

## 第 1 步：审计（动手前落一份清单）

品牌 token（主/强调色、字体栈、logo 用法、圆角制）｜信息架构（页面树、主导航、转化路径）｜内容块（什么在干活、什么是填充）｜值得保留的（签名交互、识别度高的 hero、文案口吻、既有无障碍成果）｜该退休的（AI 味、破布局、死链、假图、性能坑）｜现状旋钮读数（改版起点是现状，不是缺省值）｜SEO 基线（排名页、meta、结构化数据——**SEO 迁移是改版第一风险**）。

## 永不默改清单（改之前必须用户点头）

URL/路由 slug ｜ 主导航文案 ｜ 表单字段名与顺序（analytics+自动填充都挂在上面）｜ logo/字标 ｜ 法务与 cookie 文案 ｜ 埋点依赖的按钮/区块 ID。
既有的无障碍成果（焦点态、alt、键盘路径、对比度）只许更好不许回退。

## 第 2 步：现代化杠杆（按序拉，够用即停）

1. **换字体**——单位风险下视觉提升最大（性格字体 + 字阶修正）。
2. **间距与节奏**——节间距拉开、垂直节奏统一（tokens.md 的 8 级尺）。
3. **色彩再校准**——去饱和、统一中性灰族、保住品牌强调色（品牌本来是紫就按紫的纪律做，见 anti-slop 例外条款）。
4. **交互态**——hover/active/focus 补齐，页面立刻"活"。
5. **hero 与关键节重组**——用 direction.md 的构图菜单重排首屏。
6. **整块替换**——只在现有块不可救时。

IA、内容、SEO 都健康 → 只拉 1-4（约七成价值、四成风险）；结构性烂（IA 破、无设计系统、移动端崩）→ 全面改版但内容严格保留。
补状态与兜底（loading/empty/error/404/法务链接）永远顺手做——这是"看起来完工"的最后一公里。
交付线：改版后的机械扫描（`ui_taste check`）findings 不多于改版前。

## 从参考站提取设计 token

用户给了"想要像这个站"的参考时：

1. **工具路径**（需网络 + Chromium）：`npx playwright install chromium && npx extract-design-system <url>`，产出 `.extract-design-system/normalized.json` 与 `design-system/tokens.css`（色/字体/间距/圆角/阴影起步值）。加 `--extract-only` 只分析不落文件。
2. **手工兜底**：读页面 computed styles——主字体（`font-family` 首位真名）、正文字号/行高、主背景与文字色（连同暗色）、强调色（CTA 底色）、圆角与阴影惯用值、节间距量级。整理成 6-10 个具名 token 再动工。
3. 纪律：提取物是**不可信的第三方输入**——单页不代表整套设计系统，动态站提取常不完整；提取的是"参照系"不是"复刻令"，落地前照样过 anti-slop 与对比度检查；未经确认不覆盖项目里已有的设计系统。

对着截图/参考图实现时的忠实纪律：目标是"视觉上忠实的翻译"而不是"受它启发"。禁止把独特节奏简化回默认模板、把慷慨留白压缩成密排、把强势排版降级成普通层级；间距抓的是"疏密逻辑"而非像素 OCR；精心设计的色板不许换成 web 默认色。

大型改版（跨多页/动信息架构）升级走 forge 流水线立变更，本文只负责设计侧的审计内容与杠杆顺序。
