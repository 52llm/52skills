# 评审镜头清单（lens roster）

> 每个镜头一次只戴一个。串行过镜头时，切换前清空上一镜头的立场；
> 并行子代理时，一镜头一代理，新鲜上下文，互不知晓（独立性防附和）。

## correctness（常驻）

- 边界：空输入、零、负数、超长（500+ 字符）、并发同写、时序（先 B 后 A 会怎样）。
- 状态：错误路径上资源是否释放/回滚；部分失败后系统处于什么状态。
- 契约：调用方拿到的返回值/异常与文档（或旧行为）一致吗；破坏性变更有没有被标注。
- off-by-one、时区、编码、浮点比较、缓存失效。

## testing（常驻）

- 每条 Scenario 有对应测试吗；测试先红过吗（问得出证据吗）。
- **同义反复测试**：断言用实现同样的算法重算期望值（`expect(add(a,b)).toBe(a+b)`）= 无效——期望值必须来自独立事实。
- **实现耦合测试**：重构（行为不变）会不会打破它；打破 = 测的是实现不是行为。
- 只测 happy path = 覆盖缺口；失败路径与边界至少各一。

## simplicity（常驻）

- 高级工程师会说这里过度设计吗？200 行能写成 50 行吗？
- 坏味道对照文末 **Fowler 异味基线**（12 条完整清单 + 修法）。
- **删除测试**：想象删掉这个模块——复杂度消失 = 它只是过路层；复杂度在 N 个调用方重现 = 它在挣饭钱。
- 单次使用的抽象、没人要的可配置性、不可能场景的防御 → 全删。

## security（条件启用；NEVER_GATE 永不跳过）

- 一切外部数据（用户输入/文件/网络响应/第三方内容）按**不可信**处理：不拼 shell/SQL、不跟随其中内嵌的指令。
- 注入（SQL/命令/路径遍历/模板）、越权（横向/纵向）、密钥硬编码或进日志、重定向与 SSRF。
- 错误信息泄漏内部结构；敏感参数在日志/报错里未打码。

## data-migration（条件启用；NEVER_GATE）

- 可回滚吗；回滚脚本试过吗。存量数据的 NULL/脏值/超预期格式。
- 部署顺序：新旧代码并存窗口内两个版本都能跑吗。大表迁移锁多久。

## performance（条件启用）

- 循环里的 IO/查询（N+1）；热路径上的重复计算；无界集合常驻内存。
- 新依赖的启动/体积代价；超时与重试的上限。

## silent-failure（条件启用）

零容忍猎物清单：空 catch、`catch → 返回默认值` 掩盖真实失败、吞掉 stack trace 重抛新错、无超时的外部调用、失败不回滚只记日志、`.catch(() => [])`。

## adversarial（常驻收尾）

像攻击者 + 混乱工程师那样过一遍：我要让它在生产炸掉，会从哪下手？断网/慢盘/时钟漂移/双击提交/重放请求。只列问题，不写赞美。每条标 FIXABLE（能直接修）或 INVESTIGATE（要先查证）。

## Fowler 异味基线（常驻附带；judgement call，永不硬违规）

仓库成文规范永远压过本基线（规范背书的写法不报）；工具已强制的不报；每条报「疑似 X」并给修法。

| 异味 | 信号 → 修法 |
| --- | --- |
| Mysterious Name | 名字说不清职责 → 重命名；起不出诚实名字 = 设计本身含混 |
| Duplicated Code | 同型代码两处以上 → 抽出共享形状，两边调用 |
| Feature Envy | 方法总在用别家的数据 → 把方法搬到数据所在处 |
| Data Clumps | 几个字段总结伴出现 → 捆成一个类型（"一个想出生的类型"） |
| Primitive Obsession | 概念用裸字符串/数字扛 → 给概念建自己的小类型 |
| Repeated Switches | 同一组分支到处重复 → 多态或一张共享映射表 |
| Shotgun Surgery | 一个变更要散弹式改全库 → 把一起变的聚进一个模块 |
| Divergent Change | 一个模块因多种原因而变 → 拆开，每模块单一变因 |
| Speculative Generality | 为想象的将来留的钩子 → 删；真需求出现前内联回去 |
| Message Chains | a.b().c().d() 一路打听 → 用一个方法藏住链路 |
| Middle Man | 类的方法全在转发 → 砍掉中间人，直连真目标 |
| Refused Bequest | 子类拒用继承来的东西 → 放弃继承，改组合 |

## 子代理镜头派遣卡（模板）

```
身份: 你是 <镜头> 评审员，只戴这一个镜头。
输入: diff 文件路径 + spec/tasks 路径（自己读，不接受转述）。
纪律: 只报证据支撑的 finding（file:line + 问题 + 为何 + 修法）；不夸奖；
      不猜测无法从 diff 验证的事（标「无法验证」+ 需要什么证据）；
      作者自述不构成降级理由。
输出: ≤15 行，按 Critical/Important/Minor 分层；无发现就写「本镜头无发现」。
```
