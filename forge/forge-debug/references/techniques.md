# 排障技法箱

## 根因回溯（trace back）

报错处 ≠ 出错处。从症状沿链路往回走：

1. **数据流回溯**：坏值是谁传进来的？逐层 print/断点，直到看到"这一层输入还是好的，输出坏了"。
2. **调用栈插桩**：在可疑构造点埋 `new Error().stack` / `traceback.format_stack()`，弄清"到底是谁在什么路径上调了它"。
3. **时间回溯（git bisect）**：行为以前是好的 → `git bisect run <复现命令>` 二分出引入 commit，读那个 diff 而不是读全库。
4. **污染源二分**：测试单跑绿、合跑红 = 有测试污染共享状态。二分测试集合（跑前半 vs 后半）定位污染者，成本 O(log n)。

## 复现率放大（非确定性 bug）

- 循环跑：`for i in $(seq 100); do <cmd> || break; done`，量化复现率再对比改动前后。
- 放大条件：并发数调高、延迟注入（人为 sleep）、缩小池子（连接池=1 逼出竞争）。
- 固定随机性：种子、时间 mock、排序稳定化——先让它确定性失败，再修。

## 条件等待替代 sleep

测试里 `sleep 2` 等异步结果 = 慢且脆。换成轮询条件：

```
waitFor(条件, 超时) { 每 50ms 查一次条件，成立即返回，超时抛出并带上下文 }
```

等的是**状态**不是时间；超时报错要说清"等什么没等到"。

## 纵深防御（修完之后想一层）

同类 bug 结构性免疫的四层，按性价比挑 1-2 层做：

1. **入口校验**：非法值进不来（类型/范围/格式在边界拒绝）。
2. **业务断言**：不变量写成断言，坏状态当场炸而不是漂到下游。
3. **环境守卫**：配置/依赖缺失启动即失败（fail fast），不要跑到一半才发现。
4. **调试插桩**：这次排障用的日志点，有价值的转正（降级为 debug 级别留下）。

## 常见自欺（排障版借口拦截）

| 念头 | 现实 |
| --- | --- |
| 「多半是 X，先改了再说」 | 那是假设不是结论。写进竞争假设表，验了再动 |
| 「复现太麻烦，直接看代码吧」 | 没有红色回路 = 修完也不知道修没修好 |
| 「加个 try/except 先兜住」 | 兜住 = 把尸体埋了，坟头以后更难找 |
| 「改完这次应该就好了」（第 3 次） | 3 次失败 = 层位错了。停，回熔断协议 |
| 「就本地环境有问题，不管了」 | 环境差异本身就是根因线索 |
